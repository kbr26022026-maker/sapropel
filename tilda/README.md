# Эко Органика — версия для Tilda

Полная переработка React-сайта под Tilda. Каждая секция — отдельные файлы `HTML / CSS / JS`, готовые к вставке в блоки Tilda.

## Как подключать (делается один раз)

1. Откройте **Настройки сайта → Ещё → HTML-код для вставки внутрь HEAD всех страниц**.
2. Скопируйте туда содержимое файла [`_global/head.html`](_global/head.html) — это шрифт Inter, meta-теги и CSS-переменные темы.
3. В **Настройки сайта → Ещё → HTML-код перед закрывающим тегом `</body>`** вставьте содержимое [`_global/global.js`](_global/global.js), завернув в `<script>...</script>` — это IntersectionObserver для анимаций появления.

## Как подключать секции

Для каждой секции:

1. Добавьте на страницу блок **T123 (Прочее → HTML-код)**.
2. Вставьте внутрь блока:
   ```html
   <style>
     /* ...содержимое файла section.css... */
   </style>

   <!-- ...содержимое файла section.html... -->

   <script>
     /* ...содержимое файла section.js... (если файл есть) */
   </script>
   ```
3. Сохраните блок и опубликуйте страницу.

## Особенности реализации

- **Namespace `eo-*`** — все классы начинаются с `eo-`, не конфликтуют с `t-*` / `tn-*` Tilda.
- **Низкая специфичность** через `:where(.eo-...)` — Tilda не перебьёт стили.
- **Нет внешних зависимостей** — все иконки инлайн SVG, анимации на чистом CSS + IntersectionObserver.
- **CSS Custom Properties** — поменяйте цвет темы в одном месте (`_global/head.html`).
- **Адаптив** через `clamp()` и CSS Grid без жёстких брейкпоинтов.
- **Форма заявки** — это визуальная заготовка. В проде подмените её на штатный блок формы Tilda (`T830`/`T840`) или пропишите обработчик в `08-contact/contact.js` на ваш endpoint.

## Быстрый старт

1. Создайте новую страницу в Tilda.
2. Подключите `_global/head.html` и `_global/global.js` (шаги из первого раздела).
3. Добавляйте блоки T123 по порядку из таблицы выше, копируя файлы секций.

## Блог и статьи (папка `10-articles/`)

Блог нужен для улучшения SEO-индексации: регулярно обновляющиеся тематические материалы попадают в поиск и приводят трафик по длинным низкочастотным запросам. В проекте лежат два готовых шаблона в `eo-*` стиле: хаб со списком статей и страница одной статьи с полной типографикой и `schema.org/BlogPosting`.

### Состав

| Файл | Назначение |
|---|---|
| `10-articles/list.html` | Секция «Статьи и новости» — hero + сетка карточек + фильтр по категориям |
| `10-articles/list.css` | Стили хаба |
| `10-articles/article.html` | Шаблон страницы одной статьи — breadcrumbs, заголовок, обложка, контент, CTA, «Читайте также», JSON-LD |
| `10-articles/article.css` | Стили статьи (типографика h2/h3/p/ul/blockquote/img и т. д.) |
| `10-articles/teaser.html` | Teaser-блок «Статьи и новости» для главной — заголовок, 3 последние статьи, кнопка «Все статьи». Ведёт на `/articles/` |
| `10-articles/teaser.css` | Стили teaser-блока |

Превью:
- `preview-articles.html` — список статей.
- `preview-article.html` — одна статья с тестовым контентом.

### Вариант А — статичный блог (минимум настройки)

Подходит, если статей немного (до ~20) и писать их будете не часто.

1. В Tilda создайте страницу с URL `/articles/` → добавьте блок T123 → вставьте `10-articles/list.html` + `10-articles/list.css` (по схеме из раздела «Как подключать секции»).
   На **главной странице** дополнительно добавьте блок T123 с `10-articles/teaser.html` + `10-articles/teaser.css` (между «Статьи и документы» и «Контакты») — он даёт inbound-ссылку на `/articles/` и три якоря на последние статьи (важно для SEO).
2. Для каждой статьи:
   - Создайте в Tilda страницу с URL `/articles/slug-statyi/`.
   - В настройках страницы заполните `Title`, `Description`, `OG:image` (это Tilda индексирует автоматически).
   - В блок T123 вставьте `10-articles/article.html` + `10-articles/article.css`.
   - Замените в скопированном HTML:
     - `<h1 itemprop="headline">` — заголовок статьи.
     - Дату в `<time datetime="…">`.
     - Обложку `<img src="…">`.
     - Категорию `<span class="eo-article__cat">…</span>` и в breadcrumbs.
     - Текст внутри `.eo-article__body` (пишите обычным HTML — h2/h3/p/ul/blockquote/figure).
     - В конце — JSON-LD: поля `headline`, `description`, `image`, `datePublished`, `mainEntityOfPage.@id` привести к вашей статье.
3. В хабе (`list.html`) добавьте новую карточку в `.eo-articles__grid` — скопировать любой `<li>`-элемент, подправить ссылку, обложку, категорию и текст.

### Вариант Б — Tilda Feeds (Потоки) + наши стили

Подходит, если статей много или писать их будут нетехнические люди через визуальный редактор Tilda.

1. **Создать Поток**: в админке Tilda → раздел «Потоки» (Feeds) → «Создать поток» «Статьи». Настроить поля: обложка, категория, дата, автор, короткое описание.
2. **Настроить URL** потока: `/articles/` для списка, `/articles/<slug>/` для каждой записи (в настройках потока → «Адреса страниц»).
3. **Подключить глобальные стили**: файлы `_global/head.html` уже вставлены в Настройки сайта → HEAD, ничего дополнительно делать не нужно — `eo-*` стили будут доступны на всех страницах, включая страницы потока.
4. **Оформить список (страница хаба)**:
   - Вариант «чисто Tilda»: использовать встроенный блок Feed-списка (например, `BF501`/`BF502`/`FT101`) и стилизовать через Zero Block либо через `Tilda → Настройки блока → CSS-классы`.
   - Вариант «наш стиль»: взять `10-articles/list.html`, удалить из него элементы `<li>` в `.eo-articles__grid` и рядом (под сеткой) добавить встроенный Tilda Feed-блок. Hero-часть и фильтры останутся от нас, сами карточки будет рисовать Tilda.
5. **Оформить одну статью**:
   - Tilda создаёт страницу поста автоматически. В «Шаблоне поста потока» используйте блоки `BF701+`/`TN` или один блок T123 с `10-articles/article.html`.
   - Чтобы переменные потока подставились в наш шаблон, замените статические значения на Tilda-переменные потока (синтаксис `@field_name@` или вставка через визуальный редактор). Обязательные поля в JSON-LD: `headline`, `image`, `datePublished`, `description`.

### SEO-чеклист (применяется к обоим вариантам)

- [ ] У каждой статьи уникальный `<title>` и `<meta name="description">`. В Tilda — вкладка «Настройки страницы» → «SEO».
- [ ] Заполнены `og:title`, `og:description`, `og:image` (Tilda их генерирует автоматически из настроек страницы).
- [ ] Внутри статьи один `<h1>` (уже стоит в `article.html`), дальше только `<h2>/<h3>`.
- [ ] Все `<img>` — с атрибутом `alt` (он у шаблона есть, не удаляйте).
- [ ] `datePublished` и `dateModified` в JSON-LD реальные и в формате `YYYY-MM-DD`.
- [ ] URL статьи человекочитаемый (латиница, дефисы, 3–7 слов, без дат).
- [ ] В хабе `list.html` добавлен пункт в раздел «Читайте также» и в `.eo-articles__grid` → внутренняя перелинковка улучшает индексацию.
- [ ] Добавьте ссылку на `/articles/` в главное меню (`01-navbar/navbar.html`).
- [ ] В Tilda → «Настройки сайта» → «SEO» → подключите Яндекс.Вебмастер и Google Search Console.
- [ ] В `robots.txt` и `sitemap.xml` (Tilda генерирует сама) убедитесь, что `/articles/*` не закрыт от индексации.
